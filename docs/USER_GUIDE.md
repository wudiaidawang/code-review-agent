# CodeLens 使用指南

CodeLens 是本项目的浏览器入口。它将导入的代码放入受控临时 Git 仓库，再交给 Investigation Agent（代码调查智能体）阅读；它不会执行导入仓库中的代码。

## 打开与导入

启动服务后打开 <http://127.0.0.1:8000/>。

### 登录、注册与历史记录

首次使用时注册用户名和至少 8 位密码，之后登录。密码以 scrypt（密码哈希算法）+ 随机 salt（盐值）保存，不保存明文密码；浏览器只保存一个 14 天有效的会话令牌。对话标题、导入仓库信息与消息记录保存到服务端 SQLite（轻量数据库），查询、保存、删除均按当前用户 ID（身份标识）过滤，因此不同账号看不到彼此历史。

这是本机单机账号机制，不是互联网生产级身份系统：尚未提供邮件验证、找回密码、请求限流、HTTPS（加密传输）强制或多实例共享会话。不要在非可信网络环境复用重要密码。

你可以选择以下任一方式导入：

- **导入本地代码**：点击“导入本地代码”，选择代码文件夹。浏览器上传文件内容，不会把你的本机绝对路径交给后端。
- **从 GitHub 引入**：输入公开仓库的 `https://github.com/owner/repository` 地址。暂不支持私有仓库、SSH 地址和其他 Git 托管站。

当前导入上限为 500 个文件、50 MB。调查器主要分析 Python 源码；HTML、CSS、JavaScript 可作为前端资源被导入，但不会获得 Python AST（抽象语法树）级别的符号分析。

## 可以问什么

### 1. 代码库结构问答

这类问题走 Codebase QA（代码库问答）通道。系统先建立受限代码地图，读取至多 6 个关键 Python 文件，再由 LLM（大语言模型）基于 Evidence（证据）回答。

可直接问：

- `这个项目有几个接口？`
- `有哪些 API 路由？`
- `固定 Review Pipeline 在哪里？`
- `前端入口在哪个文件？`
- `项目的 API 模块有哪些职责？`
- `异步任务是如何组织的？`

对于接口/路由问题，系统会确定性扫描 FastAPI 装饰器，列出 HTTP 方法、路径和处理函数；不会只靠模型猜测。

### 2. 符号定位

用于找一个类、函数或方法的定义位置。

- `ReviewPipeline 定义在哪？`
- `register_routes 在哪个文件？`
- `AsyncJobManager 的实现位置在哪里？`

### 3. 实现解释

用于理解一个具体符号的行为。系统会优先取定义、实现代码和必要调用关系。

- `ReviewPipeline.run 做了什么？`
- `AsyncJobManager 如何限制并发？`
- `WorkspaceManager.prepare 为什么创建快照？`

### 4. 调用链与引用

用于追踪代码如何被调用或使用。

- `从 jobs/investigate 到 InvestigationAgent 的调用链是什么？`
- `谁调用了 ReviewPipeline.run？`
- `register_routes 注册了哪些端点？`
- `AsyncJobManager 被哪些模块使用？`

### 5. 对比与影响分析

- `review 和 investigate 两个接口有什么区别？`
- `修改 WorkspaceConfig.max_total_bytes 会影响哪些流程？`
- `ReviewPipeline 与 InvestigationAgent 的职责有什么不同？`

## 回答如何产生

普通调查遵循以下边界：

1. Planner（规划器）将问题拆为符号与关系；
2. 规则工具检索定义、代码窗口、引用与调用边；
3. Completion Gate（完成闸门）检查证据是否足以支撑回答；
4. LLM 只根据已验证 Evidence（证据）组织答案，并可申请一次受控补缺；
5. 若没有足够证据，系统会明确说明，不会用猜测补齐。

结构问答不强行套用符号合同：Codebase QA（代码库问答）会从代码地图中提供受限阅读材料；所有结论仍应对应返回中的 Evidence（证据）。

## 目前前端具备的功能

- 本地文件夹与 GitHub 公开仓库导入；
- 对话式代码调查；
- 异步 Job（任务）提交，页面显示“建立调查计划/检索证据”等状态；
- 浏览器本地保存的对话历史、新建对话、清空对话；
- 回答中的 Evidence（证据）数量提示。

服务端最多接受 50 个排队或运行中的 Job（任务），默认最多 8 个 Worker（工作槽）同时执行。该限制保护本机磁盘和模型服务；并不等于 50 个任务会同时执行。

## 当前不在前端中的功能

- **固定 Review Pipeline（审查管道）**：后端已有 `POST /jobs/review`，它针对 Git diff（代码变更）执行 `PlanBuilder（计划生成器）→ Executor（执行器）→ Aggregator（聚合器）→ Report（报告生成器）`。当前聊天界面只接 Investigation（代码调查），尚未提供“选择 base/head 后执行变更审查”的页面控件。
- **完整评测**：外挂仓库批量评测、指标聚合、LLM Judge（模型裁判）属于离线流程，不通过聊天页面执行。
- **私有 GitHub 仓库、非 GitHub 地址、仓库写入和代码执行**：当前均不支持。

## 提问建议

- 一次问一个明确目标，优先写出类/函数/文件名，例如 `解释 ReviewPipeline.run`。
- 调用链问题写清起点和终点，例如 `从 submit_investigation 到 investigate 的调用链`。
- “它”“这里”等依赖上一轮的指代目前不保证可靠；重新写出符号名最稳妥。
- 如果回答说“证据不足”，可以换成更具体的问法：`routes.py 中有哪些 POST 路由？`，或问对应符号的定义与实现。

## 常见问题

### 为什么回答会说没有证据？

这表示代码地图或符号调查没有找到能支撑该结论的受验证片段，不表示仓库一定没有相关代码。常见原因是问题过于宽泛、符号名不明确，或目标不是 Python 源码。

### 为什么“接口有几个”现在可以回答？

该类问题已由 Codebase QA（代码库问答）处理：它直接扫描 `@app.get`、`@app.post` 等 FastAPI 路由装饰器，不再要求 Planner（规划器）先把“接口”误判为某个函数符号。

### 我的对话历史保存在哪里？

历史保存在当前浏览器的 Local Storage（本地存储）中；换浏览器、清理站点数据或换设备后不会同步。
